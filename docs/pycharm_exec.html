3. PyCharm 環境の構築¶

3.1. Anaconda 実行パスの設定¶

  1. スタートメニュー①を出します

  2. スタートメニューは ガン無視 して、 環境変数 と入力します②

  3. システム環境変数の編集 をクリックします③

  1. システムのプロパティ が開くので

  2. 詳細設定 タブをクリックし ④

  3. 環境設定 ⑤

  1. ユーザー環境変数（上の窓） Path をクリック

注釈

下の窓（システム環境変数）にも Path があるので間違えないでください

  1. 新規

  2. anaconda3 を everything で検索して、フルパスをコピーペースト（Ctrl+V）、さらに末尾に \Library\bin を加える ⑨

注釈

everything での検索は anaconda3 の検索のしかた (3.1.1 章) を参照してください

  1. C:\Users\あなたの名前\AppData\Local\Programs\anaconda3\Library\bin となっていればよい（ここでは あなたの名前 が 16kcs） ⑨

  2. OK ⑩

    1. と 4. で開いた 環境変数 と システムのプロパティ 窓の OK をクリックし閉じます

  4. サインアウトします

  5. サインインします

注釈

Windows 3.1 時代から、システム環境変数はサインインしないと Windows が認識してくれないのは相変わらずです

3.1.1. anaconda3 の検索のしかた¶

  1. タスクバーの 虫眼鏡アイコン をクリックします

  1. 小文字で anaconda3 と入力し ①

  2. 検索結果、小文字の anaconda3 を 右クリック ②

  1. フルパスをクリップボードにコピー を選択

3.2. PyCharm の起動¶

  1. スタートメニュー①を出します

  2. スタートメニューは ガン無視 して、 pycharm と入力します②

  3. Pycharm Professional をクリックします ③

  1. Import PyCharm Settings 画面がでてきたら、 ◎ Do not import settings ④ OK ⑤

注釈

Import PyCharm Settings 画面はでなくても問題ないです

  1. New Project +

3.3. Python プログラムファイルを置くディレクトリと Python バージョンの設定¶

  1. Pure Pytyon をクリック ①

  2. 一番上の Location: は、Python プログラムファイルを保存するディレクトリです。変更したければ 📁 ②

注釈

よくわからなければ操作の必要はありません

  1. New environment using v Virtualenv の右 ▼ をクリック ③

  2. 〇 Conda を選択 ④

  1. 3.9 の右 ▼ をクリック ③

  2. 3.8 を選択 ⑥ 1

  1. _conda.exe を everything で検索して、フルパスをコピーペースト（Ctrl+V）します ⑦

注釈

everything での検索は _conda.exe の検索のしかた (3.3.1 章) を参照してください

  1. Create ⑧

1
ここでは使用する Python のバージョンを選択しますが最新の 3.9 のひとつ前、3.8 を選択します。本日インストールした Anaconda3-2021.05 の 4 つ前のバージョン 2020.02 では、当時最新の 3.8 もインストールできましたが、
「主要なOSSプロジェクトが 3.8 に未対応なためインストーラーは 3.7 にしていると書いてある。 → AnacondaでPython3.8を使う 」
との記述があります。2021.05 も同様で、3.9 がいろいろ追い付いていないため 3.8 にしていると思われます。 Anaconda と Pythonのバージョン対応 によると、たしかに 2021.5 は Python 3.8 を前提として作成されていることがわかります。

3.3.1. _conda.exe の検索のしかた¶

  1. タスクバーの 虫眼鏡アイコン をクリックします

  1. 小文字で _conda.exe と入力し ①

  2. 検索結果、 _conda.exe を 右クリック ②

  1. フルパスをクリップボードにコピー を選択
