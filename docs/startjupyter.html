4. JupyterNotebook を動かす¶

Python などを Web ブラウザ上で記述・実行できる環境 Jupyter Notebook を動かします。

4.1. Jupyter パッケージのインストール¶

  1. PyCharm はショートカットが異常に多いので起動のたびに Tip of the Day で紹介されます。一通り読んだら Close

  1. Python Packages をクリック

  1. 画面が分割されるので

  2. jupyter を入力します ① （未確認だけど、ここから 7. にジャンプしてもいいはず）

  3. jupyter 文字が入ったパッケージがテキトーにしかもたくさんでてくるけど上下にスクロールさせ ②

  4. jupyter そのものをクリック ③

  5. Install ④

  1. ユーザーアカウント制御 は、 はい

4.2. Python プログラムを Jupyter Notebook 上で動かしてみます¶

4.2.1. PyCharm¶

  1. 📁PythonProject をクリック ①

注釈

_conda.exe の検索のしかた (3.3.1 章) の 2 で別フォルダを指定していたら、 PythonProject とは出てきません。そのときは、一番上の階層の 📁 をクリックしましょう 1

  1. 次に File ②

  1. ずらっとでてきますが二つ目の New...

  1. Jupyter Notebook

  1. test と入力し Enter

  1. test.ipynb というファイルができているはずです

  1. test.ipynb をクリックします ①

  2. 下記 Python プログラムをコピペします ②

1
2
3
4
def say_hello(name):
    print("hello, {}!".format(name))

say_hello('hage')

  1. ▶ をクリック ③ メニューが広がります

  1. ▶Run Cell で実行しましょう

  1. Jupyter Notebook Web サーバーが立ち上がります（後ほどみてみます）

  1. hello, suzuki が実行結果です。実行したプログラムの吹き出し直下に表示されます ④

  2. Register project interpreter as a kernel をクリックしてください ⑤

警告

Register project interpreter as a kernel をクリックした瞬間に下記の画面がでてくるときは Anaconda 実行パスの設定 (3.1 章) の手順を間違えています。やり直しましょう

注釈

Windows セキュリティの重要な警告 は次のように対処します。

  1. ☑ プライベートネットワーク（ホームネットワークや社内ネットワークなど） ⑥

  2. □ パブリックネットワーク（空港、喫茶店など）（非推奨） ⑦

  3. アクセスを許可する ⑧

1

この一番上階層を、プロジェクトルートといい、プルジェクトルートの下のファイル（とディレクトリ）群でプロジェクトを構成します。プロジェクト毎に Python のバージョン、インストールするモジュールを互いに干渉させることなくインストールすることが可能です

4.2.2. Web ブラウザ¶

PyCharm (4.2.1 章)からの続きです。

  1. Jupyter をクリック ①

  2. Event Log をクリック ②

  1. 下の段が分割され、右側に Event Log が現れます

  2. Open In Browser をクリックします、ハイパーリンクになっていて Web ブラウザが立ち上がります

  1. test.ipynb をクリックします

  1. PyCharm で打ち込んだプログラムが反映されています。 ▶ Run で実行してみます

  1. 結果は PyCharm と同じになります
